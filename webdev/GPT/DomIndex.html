Preguntas

  1. ¿Qué es el DOM?
    • BingAI: El DOM (Document Object Model) es la estructura del documento HTML. Una página HTML está formada por múltiples etiquetas HTML, anidadas una dentro de otra, formando un árbol de etiquetas relacionadas entre sí, que se denomina árbol DOM (o simplemente DOM). El DOM es una interfaz de programación que nos permite crear, cambiar o remover elementos del documento.
    • Referencia
  2. ¿Qué es un nodo?
    • BingAI: En el contexto del DOM, un nodo es un único punto en el árbol de nodos. Los nodos pueden ser varias cosas el documento mismo, elementos, texto y comentarios. Dicho de manera simple, un nodo es una etiqueta HTML sobre la que vamos a poder realizar operaciones de lectura y de escritura.
    • Referencia
  3. ¿Cómo se accede a un elemento del DOM?
    • BingAI: Para acceder a un elemento del DOM se pueden utilizar los siguientes métodos:
      document.getElementById(id) document.getElementsByClassName(nombreDeClase) document.getElementsByTagName(nombreDeEtiqueta) document.querySelector(selectorCss)
    • Referencia
  4. ¿Cómo se modifica un elemento del DOM?
    • Para modificar un elemento del DOM se pueden utilizar los siguientes métodos:
      element.innerHTML = nuevoContenido
      element.attribute = nuevoValor
      element.setAttribute(atributo, valor)
      element.style.property = nuevoValor
    • Referencia
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?
    • La diferencia entre getElementById() y querySelector() es que getElementById() es mejor soportado que querySelector(). querySelector() es mejor soportado que getElementsByClassName(), pero querySelector() te da una lista de nodos estática mientras que getElementsByClassName() te da una lista de nodos en vivo. querySelector() te permite encontrar elementos con reglas que no se pueden expresar con getElementById() y getElementsByClassName(). Con getElementById(), tienes que hacer referencia a un elemento que tenga un id único. Con querySelector(), te devolverá el primer elemento que cumpla la condición que especifiques.
    • Referencia
  6. ¿Qué es un evento?
    • En JavaScript, un evento es una notificación de que alguna característica interesante acaba de ocurrir, generalmente relacionada con el usuario que navega por la página. Dichas características pueden ser muy variadas. Los eventos son la manera que tenemos en JavaScript de controlar las acciones de los visitantes y definir un comportamiento de la página cuando se produzcan.
    • Referencia
  7. ¿Qué es un listener?
    • En JavaScript, un listener es una función que se encarga de manejar un evento específico. Los listeners se pueden agregar a los elementos HTML para detectar eventos como clics de botón, cambios de estado de entrada y más. Cuando se produce un evento, el navegador llama a la función del listener correspondiente.
    • Referencia
  8. ¿Qué es un callback?
    • En JavaScript, un callback es una función que se pasa como argumento a otra función y se invoca cuando se completa una tarea. Las funciones callback son ampliamente utilizadas en la programación con JavaScript. Puede tratarse, por ejemplo, de una función que se ejecuta al pulsar un botón o de algún contenido relacionado con una función setTimeout(), que provoca un retraso de tiempo.
    • Referencia
  9. ¿Qué es un objeto de evento?
    • En JavaScript, un objeto de evento es un objeto que contiene información sobre un evento que acaba de ocurrir. Cuando se produce un evento en un elemento HTML, como hacer clic en un botón, se crea un objeto de evento. Este objeto contiene información sobre el evento y sus propiedades.
    • Referencia
